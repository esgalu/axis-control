# Regenerate PWA home-screen icons from an SVG logo

This app's home-screen icons (`public/assets/icons/*.png`) were generated by screenshotting the logo SVG rendered inside a plain HTML page at exact pixel dimensions — not with an image library, since none is installed and adding one just for icon generation isn't worth the dependency.

## Prerequisites

A local Chrome/Chromium binary and `playwright-core` (installed ad-hoc in a scratch directory, not as a project dependency — this is a one-off build tool, not part of the app).

## Steps

1. **Build a tiny HTML harness** in a scratch directory (not inside the repo) that inlines the target SVG centered on a white square `<div>`, with the logo's width/height as a percentage controlled by a `?logoPct=` query param (so the same harness serves every size/padding combination):
   ```html
   <div id="canvas" style="width:100vw;height:100vh;background:#fff;display:flex;align-items:center;justify-content:center;">
     <svg id="logo" ...the actual logo markup...></svg>
   </div>
   <script>
     const pct = new URLSearchParams(location.search).get('logoPct') || '76'
     document.getElementById('logo').style.width = pct + '%'
     document.getElementById('logo').style.height = pct + '%'
   </script>
   ```

2. **Screenshot at each required size**, using `playwright-core`'s `chromium.launch({ executablePath: '/Applications/Google Chrome.app/Contents/MacOS/Google Chrome' })` (or the equivalent Chrome path for the current OS) and a viewport set to exactly the target pixel dimensions (`deviceScaleFactor: 1`):

   | File | Size | logoPct | Notes |
   |---|---|---|---|
   | `apple-touch-icon.png` | 180×180 | 76 | iOS home screen; iOS rounds corners itself, ship a full-bleed square |
   | `icon-192.png` | 192×192 | 76 | Android manifest icon, `purpose: "any"` |
   | `icon-512.png` | 512×512 | 76 | Android manifest icon, `purpose: "any"` |
   | `icon-512-maskable.png` | 512×512 | ~56 | Android adaptive icon, `purpose: "maskable"` — needs **more** padding (smaller logoPct) so Android's mask shape doesn't clip it; aim for content inside the center ~66% safe zone |

3. **Save directly into `public/assets/icons/`**, overwriting the existing files.

4. **Confirm dimensions** landed correctly: `sips -g pixelWidth -g pixelHeight public/assets/icons/*.png` (macOS) or equivalent.

5. **No manifest changes needed** if filenames/sizes are unchanged — `public/manifest.webmanifest` and `index.html`'s `<link rel="apple-touch-icon">` already point at these exact paths.

## Verification

Visually inspect each PNG (read it as an image) before shipping — confirm the background is solid (not transparent, which renders as a black or white box unpredictably depending on OS), the logo is centered, and there's no clipping. Actually adding the icon to a phone's home screen to confirm the final render is the only way to be fully sure — flag that as something the user needs to check on-device, same caveat as safe-area work (see `mobile-pwa-agent`).
